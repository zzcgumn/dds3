# Task 15: Prepare PR

## Objective
Create a clear, reviewable PR for the removal.

## Steps
- Commit changes with message: `refactor(heuristic): remove legacy sorting from Moves.cpp`.
- Push branch and open PR.
- In the PR description, include:
  - Summary of changes
  - API compatibility notes
  - Link to acceptance checklist (Task 14)
  - Any perf notes (Task 10)
- Request reviewers and mark as refactor.

## Verification
- CI passes; reviewers can build and test locally without flags.

## Expected outputs
- Open PR ready for review.
